# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest (`main`) | Yes |

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in Watchdog, please report it via email to **saad.r0521@gmail.com** with the subject line `[SECURITY] <brief description>`.

Include as much of the following as possible:
- Type of issue (e.g. SQL injection, XSS, auth bypass)
- File path(s) where the vulnerability exists
- Steps to reproduce
- Proof-of-concept or exploit code (if possible)
- Impact assessment

You will receive an acknowledgement within **48 hours** and a status update within **5 business days**.

Please give us a reasonable amount of time to fix the issue before public disclosure. We aim to resolve:
- **Critical** vulnerabilities within **24 hours**
- **High** vulnerabilities within **7 days**
- **Medium** vulnerabilities within **30 days**

We will credit researchers who report valid vulnerabilities (unless you prefer to remain anonymous).
